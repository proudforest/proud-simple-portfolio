# Portfolio Website Deployment Plan

## Objective
Build and deploy a modern, data-analyst-focused portfolio website on GitHub Pages.

## Specs & Design
- **Theme:** Modern Dark Theme.
- **Typography:** JetBrains Mono.
- **Hero Section:** "Space" (Data Analyst).
- **Content:**
  - Hero Section.
  - 3 Project Showcase Cards.
  - Contact Details (Email/LinkedIn/GitHub).

## Implementation Steps

### Phase 1: Local Development
1. **Initialize Project:** Create `index.html`, `style.css`, and `script.js` in a new folder.
2. **Setup HTML Structure:** Define sections (Hero, Projects, Contact).
3. **Apply Styling:** Implement modern dark theme and load "JetBrains Mono" via Google Fonts.
4. **Create Project Components:** Build responsive grid cards for the 3 projects.

### Phase 2: GitHub Repository Setup
1. **Repository:** Create a new repository on GitHub named `portfolio` (or similar).
2. **Deployment:** Push local code to the `main` branch.

### Phase 3: Deployment
1. **GitHub Pages Configuration:** Enable "GitHub Actions" or "Deploy from branch" under Repository Settings > Pages.
2. **Verification:** Confirm deployment at the generated `github.io` URL.

## Verification
- [ ] Website loads correctly on desktop and mobile.
- [ ] Dark theme colors are applied correctly.
- [ ] JetBrains Mono font is rendered.
- [ ] 3 projects are visible and linked.
- [ ] Contact details are accessible.
- [ ] Site is successfully live via GitHub Pages.
